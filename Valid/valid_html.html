HTML Best Practices

Below are some best practices for HTML coding:
  • HTML pages should have a DOCTYPE declaration.
  • All tags should be in lowercase (e.g. no caps).
  • You should close all tags (e.g. if you use the <p>, you must use the </p> tag at the end of the paragraph).
  • Alternatively, if you are using an empty tag such as <br>, then you should write it as <br/>.
  • If you use an attribute to a tag, the value should be enclosed in quotation marks (e.g. src="image.jpg", not src=image.jpg).
  • You can nest tags, but use caution because improper nesting can result in code failure (e.g. use <b><em>text</em></b>, not <b><em>text</b></em>).
